# Code of Conduct

## Our Pledge

We want the quiCkLI community to be open, respectful, and useful for contributors with
different backgrounds and experience levels.

Participants in this project are expected to act professionally and constructively in all
project spaces, including issues, pull requests, discussions, reviews, and release
feedback.

## Expected Behavior

- Use welcoming and inclusive language.
- Give technical feedback in a factual and respectful way.
- Assume good intent, but address problems directly when behavior causes friction.
- Focus discussions on the project, the code, and the user impact.
- Respect different levels of experience and help keep the project accessible.

## Unacceptable Behavior

- Harassment, insults, or personal attacks.
- Discriminatory or exclusionary language.
- Publishing private information without permission.
- Repeatedly derailing technical discussions.
- Any conduct that would make participation unsafe or unnecessarily hostile.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, or other
contributions that do not align with this Code of Conduct.

## Reporting

If you experience or witness unacceptable behavior, open a private maintainer contact
channel before escalating publicly. If no private channel is available yet, create a
GitHub issue that requests private follow-up without including sensitive details.

## Scope

This Code of Conduct applies to the repository and to project-related public interactions.
